# PROJECT: 本地数据分析与图表展示软件

> 本文档是项目的唯一事实来源（Single Source of Truth）。所有进度、决策、风险都以这里为准。

## 1. 项目愿景
- 做一款本地运行的桌面数据分析工具，支持表格导入、数据处理、列选择、多Y折线图、时间粒度聚合、统计计算、结果导出，并在界面中完整展示原始、处理、合并数据与分析结果。
- 当前版本：V1.5（导入文件夹+路径记忆）
- 目标平台：Windows 本地桌面

## 2. 当前里程碑
- **V1.2**：已完成，归档完成
- 包含能力：
  - [x] 独立数据处理模块（条件删除整行/替换均值）
  - [x] X轴时间粒度聚合（分钟/小时/班次/天/周）
  - [x] 多文件导入与分开展示
  - [x] 多文件按时间排序合并（内存临时）
  - [x] 临时储存区管理原始/处理/合并数据集
  - [x] 切换/导出/删除临时数据集

## 4. 关键决策（ADR）
- ADR-001: Python + PySide6 + pandas + PyQtGraph
- ADR-003: 方差/标准差默认样本统计（ddof=1）
- ADR-005: 原始数据前1000行预览，统计/绘图基于全量有效数据
- ADR-006: Y轴升级为多Y多选+颜色配置
- ADR-008: DatasetManager统一管理多数据集
- ADR-009: 处理/合并结果不覆盖原始数据，统一进入临时储存区
- ADR-010: 时间粒度聚合默认取窗口均值，只影响绘图层

## 2026-07-03 Bug修复记录：UI问号乱码与时间粒度分析失败

### 问题现象
- 软件左侧/图表面板出现问号 `??`
- 切换时间粒度（分钟/小时/班次/天/周）时分析失败
- 日志中出现：
  - `ValueError: too many values to unpack (expected 2)`
  - `AttributeError: 'NoneType' object has no attribute 'setLabelText'`

### 根因分析
1. `app/ui/widgets/chart_config_panel.py` 与 `app/ui/widgets/chart_panel.py` 部分中文文案编码损坏，界面显示为问号/乱码
2. Y轴序列结构已升级为三元组 `(列名, 颜色, 是否显示均值线)`，但主窗口绘图准备逻辑仍按二元组解包，导致时间粒度切换和绘图时报错
3. 后台线程进度回调在进度对话框已销毁时未做空保护，导致偶发异常

### 修复内容
- 修复 `chart_config_panel.py` 中文文案：折线图配置、Y轴多选、均值线总开关、按钮文本、提示文案
- 修复 `chart_panel.py` 中文常量：原始/分钟/小时/班次/天/周、均值、数值、无数据提示、早晚班标签、降采样提示
- 修复 `main_window.py`：
  - Y轴序列解包兼容三元组
  - 恢复正确的时间粒度常量“原始”
  - 修复图表标题与渲染提示文案
  - 修复进度回调空对象/已销毁对象保护
- 保持多Y序列+单序列均值线控制能力不变
- 保持日志系统、后台线程分析、时间粒度聚合能力不变

### 验证结果
- `compileall` 编译通过
- 应用真实启动测试通过，不秒退
- 核心报错链路修复完成：Y序列解包、图表配置文案、进度回调异常
