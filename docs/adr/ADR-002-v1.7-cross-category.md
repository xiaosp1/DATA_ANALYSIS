# ADR-002 — V1.7 跨类别（机头/机尾）同图显示方案

- 日期：2026-07-10
- 状态：Accepted
- 决策者：尘醒（Owner）/ dataanalysis-pm

## 背景
V1.6.1 支持多文件合并，但隐含前提是"各文件由同一程序产生、列名一致、时间无重叠"，采用纵向 concat+时间排序。
现需引入**第二类文件**（命名为"机尾"，第一类命名为"机头"）：
- 两类文件分别导入（分两次导入，可同次选多个文件）
- 两类文件只有时间列列名相同（均为 `时间`），其他列列名互不相同
- 两类时间戳格式/精度相同，但时间可能重叠、也可能有先后
- 两类各有一个**单像素精度因子**（由用户手动输入，一类仅对应 1 个 factor）
- 目标：同一个折线图中同时显示两类文件中用户选定的 Y 列

## 关键决策

### D1. 类别命名
- 两类类别名固定为：`机头`（第一类）、`机尾`（第二类）
- 内部 code 使用 `head` / `tail`，UI 显示 `机头` / `机尾`

### D2. 时间列识别
- 两类文件时间列列名固定为 `时间`，沿用现有自动识别逻辑
- 自动识别失败时保留现有"让用户选 X 轴列"的兜底

### D3. 跨类对齐策略 = outer join（独立序列叠加）
- 同一类别内部的多文件：沿用旧逻辑（concat + 按时间排序），得到 `head_merged` / `tail_merged`
- 跨类合并：`head_merged` 和 `tail_merged` 按 `时间` 列做 **pandas outer join**（`how='outer'`）
- 不做容差/最近邻对齐（两类时间戳一致，业务上无需强行拼行）
- 合并后的非时间列加类别前缀：`[机头]原列名` / `[机尾]原列名`；已带 `(mm)` 后缀的保持在后面：`[机头]脱模力(mm)`
- 同一时刻一侧有值另一侧无值时，缺失侧保持 NaN，绘图时折线自然断开/不连线，不做前向/后向填充

### D4. 单像素精度（mm 缩放）
- 每个类别有一个独立 factor：`head_pixel_factor` / `tail_pixel_factor`，用户手动输入（>0）
- 同一类别内的所有文件共用同一个 factor（导入时输入，后导入的同类别文件默认沿用已存 factor，允许修改）
- 缩放范围扩展："当前数据集"/"所有机头文件"/"所有机尾文件"三选一
- 缩放沿用 V1.6.1 float32 单精度语义、自动跳时间列/文本/布尔、(mm) 后缀、重名去重、排除列能力

### D5. 数据集元数据扩展
- `DatasetItem` 新增字段：
  - `category`: Optional[str] = None  # None=未分类（旧文件/单类使用）；'head'/'tail'=机头/机尾
  - `pixel_factor`: Optional[float] = None  # 导入时由用户输入的该文件对应类别的 factor
  - `scaled`: bool = False  # 是否已按 factor 执行 mm 缩放，防重复

### D6. UI 改造点
- 导入："导入机头文件"/"导入机尾文件"两个入口（替换/补充原有"导入文件/文件夹"），弹出对话框让用户确认 factor
- 数据集列表：每条数据集前加类别徽标（机头=蓝色 / 机尾=橙色）
- 数据处理面板："缩放数值为 mm" 的"作用范围"新增"所有机头"/"所有机尾"选项
- Y 列选择面板：激活跨类合并数据集时，Y 列显示带 `[机头]`/`[机尾]` 前缀，多选；颜色/均值线独立控制
- 图表面板：沿用多 Y 折线；时间粒度聚合对跨类合并数据集按列分别 mean，缺失窗口为 NaN；图例显示带前缀列名
- 分析模式"折线趋势"/"描述统计"/"时序监控"全部支持跨类合并数据集（描述统计对所有数值列生效，包括带前缀列）

### D7. 临时储存区
- 同类别合并数据集命名：`机头_合并` / `机尾_合并`
- 跨类合并数据集命名：`机头+机尾_跨类合并`
- 三者都进入原有临时储存区，支持切换/导出/删除，不覆盖原始文件

## 不做什么（V1.7 范围外）
- 不做类别自定义命名（后续可做）
- 不做容差时间对齐/最近邻拼接
- 不做缺失值填充策略
- 不做第三类/多类别（仅机头/机尾两类）
- 不做 Phase 2 时序监控

## 风险
- 旧单类使用路径（未设置 category）必须保持 0 回归：原"导入文件/文件夹"走未分类路径，行为与 V1.6.1 一致
- 跨类合并后列数翻倍（机头 N 列 + 机尾 M 列 -1时间列），描述统计/散点矩阵面板在列数极多时需注意性能（沿用现有采样上限即可）
- factor 修改后已缩放的数据集不会自动重算，用户需重新执行"缩放"或重新合并（UI 给出提示）

## 验收口径

5. 旧"导入文件/文件夹"（未分类路径）行为与 V1.6.1 完全一致
